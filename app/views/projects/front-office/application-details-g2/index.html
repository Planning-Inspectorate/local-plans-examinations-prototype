Allows LPAs to submit a Gateway 2 application. Users upload required documents, complete a declaration, and confirm submission.

{% set appDetailsNotesHtml %}

Changes

  • Updated application structure to follow the Appeals pattern with individual upload steps
  • Merged declaration into a single page
  • Added a view-only ‘application submitted’ page
  • Added a progress indicator (for example, “You have completed 0 of 2 sections”)

Things to test

  • Whether users understand what documents they need to upload
  • Whether users need to upload multiple files at once or one file per step
  • Whether the application pattern is easy to follow
  • Whether next steps after submission are clear
  • Where users expect to go after submitting e.g. plan details page or submitted application view
  • Whether users expect to save progress and return later
  • Whether users need to upload additional (optional) documents
  • Whether users need to add comments or supporting information alongside uploads
  • Whether users need to access or download their submitted application
  • Whether users understand what happens when they submit (for example, can they edit after submission)

Open questions

  • What documents are mandatory for each Gateway
  • If there are no mandated documents, could other options work such as one multi-file upload + tagging journey
  • Whether document requirements differ by plan type or complexity
  • How validation should work for missing or incorrect documents
  • How multiple users accessing the service should be managed
