Notes:

  • 'My plans' LPA dashboard page created as the landing page after sign-in
  • Designed to give LPAs a quick overview of all plans and their current status
  • Table includes reference number, plan title, type, current stage and status. Need to find out if we need the 'type' column given all will be 'Local plans' for now
  • Status column aims to help users quickly understand progress and identify next actions
  • Each plan links to a detailed "Plan details" page with full visibility of stages, materials and decisions
  • Focus on supporting user needs around understanding readiness, progress across gateways, and stop/go decisions

Things to test:

  • Do users understand what each status means (for example "New", “In progress”, “With case officer”, “Completed”)?
  • Can users quickly identify which plans need attention?
  • Is the difference between “Current stage” and “status” clear?
  • Do users know what happens when they click into a plan?

Open questions:

  • Should the dashboard highlight overdue or at-risk plans more prominently?
  • Is additional filtering or sorting needed (for example by stage or status)?
  • Do users need a clearer indication of “next action” directly in the table?
