{# ===================== VERSION 1 NOTES (DEFINE FIRST) ===================== #} {% set v1NotesHtml %}

Prototype states

  • Ready
  • In progress
  • With PINS
  • Action needed

What this prototype explores

  • Gateway stages shown in a tasklist with clear statuses
  • Key metadata like dates, descriptions and actions
  • Reducing uncertainty around whether users can proceed

Key questions

  • Do users understand what each status means?
  • Is it clear why some stages are blocked?
  • Can users identify their next action?
  • Do users understand the role of PINS?
  • Is progression through gateways intuitive?

Open questions

  • LPA linking in joint plans
  • Handling missing or incomplete submissions
  • What happens when deadlines are missed

Other variations not brought to user testing:

  • Riskier option with bespoke timeline component
  • Tasklist + guidance in details component
  • Accordion-based version
